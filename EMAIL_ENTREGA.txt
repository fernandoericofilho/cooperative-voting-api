## O Que Entregamos

Desenvolvemos uma **API REST completa** para gerenciar votações cooperativas, construída com Spring Boot 3.x e Java 17. A aplicação é robusta, bem testada e pronta para ser deployada em qualquer ambiente.

### Funcionalidades Principais

A API oferece 6 endpoints que cobrem todo o fluxo de votação:

- **Criar Pauta**: Inicie uma nova votação com título e descrição
- **Listar Pautas**: Consulte todas as votações (com suporte a paginação na v1)
- **Abrir Sessão**: Configure a duração da votação (padrão: 60 segundos)
- **Registrar Voto**: Membros votam SIM ou NÃO
- **Apurar Resultado**: Veja quem ganhou (APROVADA, REPROVADA ou EMPATE)
- **API v2**: Versão simplificada para integrações diretas (demonstração de versionamento)

Todos os endpoints validam os dados, tratam erros com mensagens claras e retornam respostas em JSON.

### Como Usar

**Opção 1: Docker (Recomendado)**
```bash
docker-compose up --build
```
Tudo sobe automaticamente: aplicação em http://localhost:8080, banco de dados, migrations.

**Opção 2: Local**
```bash
# Precisa de Java 17+
./gradlew bootRun
```

**Testar a API:**
- **Swagger/OpenAPI**: http://localhost:8080/swagger-ui.html (interface interativa)
- **Postman**: Importe a collection em `docs/postman/cooperative-voting-api.postman_collection.json`
- **cURL**: Veja exemplos em `docs/TESTS.md`

### O Que Faz a Qualidade

Não é só teste: implementamos várias camadas de segurança e qualidade:

- **Validação de dados**: CPF, opções de voto, datas de sessão
- **Tratamento centralizado de erros**: 8 tipos de exceção customizados com mensagens claras
- **Masking de dados sensíveis**: CPF aparece como XXX.***.**-XX nas respostas
- **Resiliência**: se o serviço externo de elegibilidade cair, a API falha seguro (não aceita votos inválidos)
- **Logging estruturado**: cada operação é registrada para debug e auditoria
